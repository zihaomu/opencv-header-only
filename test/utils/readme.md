# `test/utils` 目录规划

## 目录职责

提供测试基础设施（数据加载、断言辅助、误差比较、路径工具等），避免业务测试重复造轮子。

## 现状

- 已有 `mat_load.*`，用于读取 `npy` 测试数据。
- 工具能力仍偏少，后续可扩展统一断言与数据构造辅助。

## 阶段计划

### P1：统一工具入口

- 抽象通用加载函数（`Mat` + 维度/类型检查）。
- 统一数据路径解析，减少硬编码路径。

### P2：断言与比较工具

- 新增数值比较工具（绝对误差/相对误差）。
- 新增形状、类型、通道一致性检查函数。

### P3：数据生成辅助

- 将 Python 生成脚本的输出规范化。
- 保证生成结果可复现并可追踪来源脚本版本。

## 完成定义（DoD）

- `test/core`、`test/imgproc`、`test/imgcodecs` 可复用同一套工具。
- 新增测试无需重复实现数据加载与误差比较逻辑。
