# `test/core/test_data/data` 数据目录规划

## 目录职责

存放 `core` 模块测试用的静态 `npy` 数据，作为回归基线输入/输出。

## 数据管理规则

- 文件名必须包含：算子名 + case 描述 + 输入/输出标识（如 `_i/_o`）。
- 新增数据必须同步更新对应测试和生成脚本。
- 不手工改写二进制数据，统一由脚本生成。

## 发展方向

- 逐步减少冗余数据，按“必要最小集 + 边界集”组织。
- 对大规模数据集采用可选下载或生成策略，避免仓库膨胀。

## 完成定义（DoD）

- 目录中每个数据文件都能定位到生成脚本与消费测试。
