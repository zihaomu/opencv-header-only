# `test/core/test_data` 目录规划

## 目录职责

管理 `core` 测试的数据生成脚本和数据清单，保证测试数据可复现、可追踪。

## 结构约定

- 生成脚本：`*_generater.py`
- 数据清单：`data/gemm_cases_manifest.txt` 等
- 二进制数据：`data/*.npy`

## 阶段计划

### P1：可复现

- 每个脚本写明随机种子、输入分布和输出语义。
- 统一脚本入口，避免不同脚本输出风格不一致。

### P2：可追踪

- 数据文件命名统一，包含算子名与 case 类型。
- 关键数据集建立版本说明（来源脚本、生成时间、阈值）。

## 完成定义（DoD）

- 删除任意数据文件后，可通过脚本重新生成并通过回归测试。
