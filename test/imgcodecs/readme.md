# `test/imgcodecs` 目录规划

## 目录职责

验证 `imgcodecs` 的读写正确性、格式支持边界和异常处理行为。

## 阶段计划

### P1：最小链路验证

- `imread`/`imwrite` 基础 round-trip 测试。
- 覆盖 `png/jpg` 主路径。

### P2：异常与边界

- 路径不存在、格式不支持、空图像等失败场景。
- 通道数与类型约束检查。

### P3：兼容性回归

- 跨平台路径与文件系统行为验证。
- 与 OpenCV 结果做关键 case 差分。

## 测试数据策略

- 小尺寸固定图片入仓库，便于快速回归。
- 大图片只在可选性能测试中使用，避免拖慢单测。

## 完成定义（DoD）

- `imgcodecs` 已公开 API 都有成功和失败路径测试。
