# `include` 目录规划

## 目录职责

`include` 是项目最终对外发布的头文件根目录。  
用户只需把该目录加入 include path 即可使用公开 API。

## 结构约定

- `include/cvh/`：主命名空间与模块入口。
- `include/cvh/3rdparty/`：必要的第三方头文件依赖（需可审计和可替换）。

## 发展原则

- 公开接口稳定优先，内部实现细节通过 `detail` 或 `*.inl.h` 隔离。
- 任何新增公开头都必须配套文档、测试或示例。

## 完成定义（DoD）

- 头文件结构满足安装与发布场景，不依赖源码目录。
