# `src` 目录规划（过渡实现区）

## 目录职责

`src` 在当前项目中仅用于迁移过渡，不是长期主实现区。  
项目最终目标是 header-only，因此长期实现应进入 `include/`。

## 管理规则

- 新增功能优先写在 `include/`（header-only 形态）。
- `src` 仅允许：
  - 迁移期兼容实现
  - 暂时无法头文件化的可选后端
- 每次迁移都要同步更新对应目录文档和删除计划。

## 阶段计划

### P0：目录收敛

- 盘点 `src/core` 文件并分类（迁移/重写/删除）。

### P1：持续迁移

- 逐步将主线能力迁移到 `include/cvh`。
- 减少默认构建对 `src` 的依赖。

### P2：退场

- 将 `src` 缩减为可选实现或完全退出主流程。

## 完成定义（DoD）

- 默认用户场景不需要 `src` 目标即可使用公开 API。
